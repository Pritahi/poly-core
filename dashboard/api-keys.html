Setup & API Keys

3 steps to connect your repo with Falsky

STEP 1 — GET YOUR API KEY

Your Falsky API key authenticates your CI workflow. Copy this key — you'll add it to GitHub as a secret in Step 2.

Loading…

⚠ Keep this secret — don't commit it to your repo

STEP 2 — ADD KEY TO GITHUB

Go to your GitHub repo → Settings → Secrets and variables → Actions → New repository secret. Name it FALSKY_API_KEY and paste your key as the value.

STEP 3 — ADD WORKFLOW YML FILE

Create this file in your repo at .github/workflows/falsky.yml. This is the complete workflow — it runs your tests and sends results to Falsky.

⚡ WHAT TO CHANGE IN THIS YML
TEST PATH Change pytest tests/test_real.py → your actual test path
PYTHON VER Change "3.11" → your project's Python version
DEPS INSTALL If using Node → replace pip install with npm ci && npm test
NO CHANGE api-url, api-key, repo-name — auto-handled ✅
Complete workflow — falsky.yml 
# .github/workflows/falsky.yml
name: Falsky Test Analysis
on:
  pull_request:
    branches: [main]
  push:
    branches: [main]

jobs:
  test-and-analyze:
    runs-on: ubuntu-latest
    env:
      CI: true
      CI_RUN_NUMBER: ${{ github.run_number }}

    steps:
      - uses: actions/checkout@v4

      - # ── Install your project dependencies ──
        name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.11"  ← CHANGE to your Python version

      - name: Install dependencies
        run: |
          python -m pip install --upgrade pip
          pip install -r requirements.txt

      - # ── RUN YOUR TESTS — change path to match your project ──
        name: Run pytest suite
        run: |
          pytest tests/ ← CHANGE: your test folder/file path \
            --junitxml=/tmp/test-results.xml \
            -v || true

      - # ── FALSKY analyzes results and posts to dashboard ──
        name: Analyze with Falsky
        uses: ./falsky-action
        with:
          junit-xml-path: '/tmp/test-results.xml'
          api-url: 'https://poly-core-vercel.vercel.app'
          api-key: ${{ secrets.FALSKY_API_KEY }}  ← from Step 2
          repo-name: ${{ github.repository }}  ← auto-detected
          comment-on-pr: 'true'
          fail-on-flaky: 'false'
          flaky-threshold: '50'
TEST FILES — WHERE DO THEY GO?

Your tests live in your own repo. Falsky doesn't decide what tests to run — it runs whatever tests you already have and then analyzes the results. Here's how pytest finds your tests:

📁 Recommended structure
your-project/
├── .github/workflows/
│   └── falsky.yml        ← workflow
├── tests/                  ← test folder
│   ├── test_homepage.py
│   ├── test_api.py
│   └── test_login.py
├── src/
│   └── app.py           ← your code
└── requirements.txt     ← dependencies
🔍 How pytest finds tests

File naming: Files named test_*.py or *_test.py

Class naming: Classes starting with Test*

Method naming: Methods starting with test_*

In YML: pytest tests/ → runs all test files in tests/ folder

NODE.JS / JEST USERS

If you're using Jest instead of pytest, replace the Python steps in your YML with these: 

      - name: Install dependencies
        run: npm ci

      - name: Run Jest with JUnit output
        run: npm test -- --ci --reporters=default --reporters=jest-junit
        env:
          JEST_JUNIT_OUTPUT_DIR: /tmp
          JEST_JUNIT_OUTPUT_NAME: test-results.xml

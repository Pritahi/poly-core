How Falsky works — setup, algorithm, API reference

01 — HOW FALSKY WORKS

Falsky is an AI-powered flaky test detector. It uses Bayesian statistics (Beta-Binomial conjugate prior) to calculate a trust score for each test — 0 to 100. The more consistently a test passes, the higher its trust score. When a test's pass rate fluctuates, Falsky detects it as flaky and categorizes the root cause.

Who decides tests? You do. Falsky runs whatever tests you already have in your repo and analyzes their results. It doesn't create or modify tests — it observes them over time and calculates reliability patterns.

02 — TRUST SCORE ALGORITHM

Each test gets a score from 0–100 based on four weighted components plus an optional flaky penalty. The core is Bayesian — Beta-Binomial conjugate prior with MAP (Maximum A Posteriori) estimation.

55% Pass Rate

Bayesian pass rate with Beta(2,2) prior. Primary reliability signal — MAP estimate adjusts for small sample sizes.

20% Recent Trend

Compares last 5 runs vs previous 5. Improving tests get bonus points, degrading tests get penalized.

10% Duration Stability

Z-score anomaly detection on execution times. Consistent timing = stable. High variance = suspicious.

10% Error Consistency

Consistent error messages = real bug. Varied/random errors = flaky behavior. Pattern hashing tracks this.

−5 TO −20 Flaky Penalty

When a flaky category is detected (timing, order dependency, shared state, non-deterministic data, environment-specific), an additional penalty is subtracted from the total score.

03 — FLAKY CATEGORIES

Falsky analyzes test history patterns to classify why a test is flaky. Each category has a distinct penalty and detection method.

⏱
Timing −8pts

High duration variance or timeout sensitivity. Z-score anomaly detection on execution times.

🔗
Order Dependency −12pts

Outcome changes based on execution order. Detected via runs test for alternating pass/fail patterns.

🤝
Shared State −15pts

Shares mutable state with other tests. Detected via clustered failure patterns and consistent error messages.

🎲
Non-deterministic −20pts

Unpredictable external data or random values. Detected via high error message diversity ratio.

🌐
Environment-specific −10pts

Pass rate differs significantly across environments (e.g. passes locally but fails in CI). Detected via environment correlation analysis.

04 — REST API REFERENCE

Write operations require the X-Poly-API-Key header. Read operations are open.
